## Decisions

- GitHub 업로드 확인 항목은 완료로 처리한다.
- 오늘 다음 우선 작업은 AWS 현재 환경 확인과 LLM 방향 결정이다.
- 실제 AWS 리소스 확인은 로컬 정보만으로 부족하므로 별도 AWS API 조회 필요 여부를 판단한다.
- 현재 환경은 AWS 계정 `580075786326` 의 `terraform-poc-iac-role` assumed role로 실행 중인 것으로 기록한다.
- 현재 AWS 환경은 `ap-northeast-2` 리전에 있으며, instance role 기반 자격 증명을 사용한다.
- S3 조회 권한이 있으므로 최소한 일부 읽기 권한은 확보된 상태로 본다.
- MVP 단계에서는 실제 LLM API 연결을 우선 검토한다.
- 무료 사용과 빠른 MVP 검증을 위해 Gemini API를 우선 provider로 본다.
- MVP의 실제 API 연결 구조는 `.env` + backend proxy 기준으로 진행한다.
- 프론트엔드는 React/Vite, 백엔드는 Express 단일 프록시 구조로 진행한다.

## Issues and Risks

- AWS 현재 환경은 아직 실제 점검하지 않았다.
- 로컬 기준으로는 `~/.aws` 설정이 없고 AWS 환경 변수도 없어 계정/리소스 식별이 불가능하다.
- 인스턴스 메타데이터 접근도 현재 환경에서는 확인되지 않았다.
- 역할은 확인됐지만 실제 접근 가능한 AWS 리소스 범위는 아직 확인하지 않았다.
- EC2 등 다른 서비스의 읽기 권한 범위는 아직 확인되지 않았다.
- 실제 API 연결 시 API key 관리, 외부 네트워크, 환경 변수 처리 방식이 필요하다.
- Gemini API를 쓰려면 외부 API 접근과 API key 보관 방식이 필요하다.
- 실제 구현 전 승인 로그에 API 접근과 `.env` 사용 항목을 반영했다.
- 아직 패키지 설치와 실행 검증은 하지 않았다.
