# Wireframe

## 1. Objective

- 보험 CS용 AI Agent CRM MVP의 화면 구조를 확정한다.
- UI 레퍼런스에서 정리한 패턴을 실제 MVP 레이아웃으로 연결한다.

## 2. Screen Goal

- 상담원이 상담 이력을 확인한다.
- 상담원이 고객 질문을 입력한다.
- AI가 답변을 생성해 우측 패널에 보여준다.
- 모든 흐름이 한 화면에서 끝나도록 구성한다.

## 3. Main Layout

- 상단: 헤더
- 중앙: 상담 이력
- 하단: 질문 입력창
- 우측: AI 답변 패널

## 5. Section Details

### Header

- 서비스명 표시
- 현재 상담 화면임을 보여주는 상태 영역
- 필요 시 `Save`, `New` 정도의 최소 액션만 유지

### Consultation History

- 시간순 메시지 표시
- `Customer`, `CS` 구분
- 복잡한 필터나 검색은 제외

### Question Input

- 고객 질문 입력 전용
- 빠르게 질문을 넣고 AI 요청할 수 있도록 단순하게 유지

### AI Agent Panel

- 현재 질문 요약
- 추천 답변
- 참고 정보 또는 주의 문구
- `Copy`, `Regenerate` 같은 최소 액션 제공

## 6. Design Direction

- 밝은 배경의 업무용 UI
- 패널과 카드 중심 구조
- 가독성 우선
- Salesforce 계열처럼 정돈된 간격과 상태 표현

## 7. MVP Rules

- 한 화면만 만든다.
- 상담 이력, 질문, 답변 외 불필요한 요소는 넣지 않는다.
- 보험 CS 문맥이 느껴지는 텍스트와 용어를 사용한다.
- 복잡한 정보 구조보다 데모 시연이 쉬운 흐름을 우선한다.
